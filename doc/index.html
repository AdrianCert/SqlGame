Abstract

SQLGame este o aplicație web creată pentru a transforma procesul anevoios şi copleşitor de a învăţa SQL într-o joacă și mult mai atractivă decât un curs clasic. Platforma este destinată pentru cei care doresc să invețe sql și tot odată reprezintă un înstrument pentru grupuri formale prin crearea de borduri. Aplicația se bazează pe întrebarări, care pot fii aduse și de către utizizatorii acestora.

Introducere

SQLGames este proiectul pe care echipa noastră l-a ales cu scopul de a pune la îndemâna oricărui elev/student o platformă de E-Learning de care să se poată folosi pentru a înțelege în profunzime limbajul SQL.

Caracteristici

Înregistrarea

Utilizatorul care nu dispune de cont, poate să-și creeze unul nou. La momentul înregistrării, li se va oferi 10-15 SQLizi. În locul SQLizilor, ar putea fi oferite 3 întrebări de nivel ușor care să poată oferi utilizatorului o ințelegere a interfaței și a metodei de validare a răspunsului.

Autentificarea

Utilizatorii care dispun de cont, pot să se autentifice și să intre pe contul lor personal.

Editarea Profilului

Fiecare utilizator poate să-și modifice profilul. Desigur, conform Politicii de Confidențialitate, ceilalți utilizator vor fi capabili sa vizualizeze numele de utilizator, rank-ul, numărul de SQLizi și imaginea de profil.

Vizualizarea Profilului

Utilizatorii vor fi capabili să își vizualizeze profilul. Ei vor fi capabili să vizualizeze istoricul în care vor fi salvate toate încercările utilizatorului, reușite cât și nereușite, fiind ordonate în ordinea efectuarii acțiunii, prima fiind cea mai recentă, ultima fiind prima actiune efectuată de către utilizator.

Vizualizarea Noutăților

Utilizatorii vor fi capabili să vizualizeze noutățile, fiind capabili să lase comentarii și să răspundă la comentariile altor utilizatori.

Interacțiunea cu Întrebări

Cumpărarea Întrebărilor

Fiecare utilizator va fi capabil să cumpere o întrebare, dacă își va permite.

Răspunderea la Întrebări

Fiecare utilizator va fi capabil să răspundă la întrebări. La răspunderea corectă la întrebare, utilizatorului i se va oferi numarul de SQLizi egal cu costul întrebării inmulțit cu 1.5, dacă întrebarea este de dificultate ușoară sau medie, cu 2, dacă întrebarea este de dificultate grea.

Crearea Întrebărilor

Fiecare utilizator va fi capabil să creeze o întrebare, dacă își va permite. Întrebările mai populare vor fi salvate, celelalte fiind șterse după trecerea unei perioade de timp.

Infrastructura

Aplicația este o aplicație scalabilă bazată pe microservicii și este alcătuită din următoarele componente:

  • Server web
  • Server api
  • Gateway
  • Naming Service
  • Programable Worker Service
  • Sql Preprocessing Service
  • Sql Processing Service

Server web

Serverul web este componenta plincipală a plicaţieri. Acesta serveşte clienţilor care se connectează pagini html și conținut static ( foi de stil, documente JavaScript, ect.). Aici se generează paginile html folosind un template engine și se livrează clienților.

Server api

Serverul api este componenta cea mai intens folosită din aplicație. Aceasta serveste raspunsuri sub forma json a cerințelor efectuate de codul JavaScript care rulează pe browserul clientului. Astefel paginile încărcate devin dinamice și se permite trecerea către o aplicație de tip WPA ( web progresive application).

Gateway

În contextul unei aplicații scalabile introducem această componentă prin care datele/ cererile sunt rutate. Dacă este nevoie de a apela la un microserviciu, cererea v-a fii preluată de către această componentă iar aceasta v-a furniza răspunsul primit de către microserviciu înapoi. Pentru scalabilitate se v-a folosii de naming services, care asigură că orice cerere primește un răspuns (exceptand cazutile excepționale când tot sistemul eșuiază).

Naming Service

Pe scurt naming services este un serviciu care monitorijează microserviciile din ecositem, echilibează sarcinile primite. Fiecare microserviciu la pornire anuntă că este online. Este nevoie de un nume unicâ prin care se identifică. Aceasta v-a fii monitorizat. Load Balancing se face prin prelucrarea în oridine a cerințelor de către un microserviciu de același tip (Rount-Robbin).

Programable Worker Service

Programable Worker Service asigură că unele acțiuni sunt efactuate la un anumit moment de timp. Această preia o cerere obisnuită și o întârzie până când se ajunge la timpul dorit( data/ oră/ înterval orar).

Sql Preprocessing Service

Interogarile Sql vor fi preprocesate pentru a stabilii corectitutinea sintatica si a de prevenii efecatuarea actiunilor rau intentionate.

Sql Processing Service

Rezultatele unei interogari sql vor fii date de catre acest microserviciu.
